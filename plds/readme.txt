1. Необходимо равномерно распределить входные данные по ниткам PLDS/THR*
2. Запускаем нулевой шаг (start_step0.cmd)
3. Ожидаем завершения
4. Запускаем первый шаг (start_step1.cmd)
5. Ожидаем завершения
6. Проверяем результат (в директориях с исходными MS продуктами появяться DRA файлы)
7. Если результаты не устраивают, корректируем цвет (открываем MS в SIP, подбираем цвет и сохраняем в палитру в исходной папке, имя файла как у продукта MS)
8. Запускаем шаг 3 (start_step3.cmd)
9. Запускаем шаг 2 (start_step2.cmd)

По завершению обработки гасим демоны шага 3
